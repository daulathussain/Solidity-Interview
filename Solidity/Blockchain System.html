🔄 Trading Flow Sequence

1
👛 Wallet Connection
User connects wallet (MetaMask, WalletConnect) and authorizes DApp access
↓
2
💰 Token Approval
User approves token spending for the exchange smart contract
↓
3
📊 Market Data
Frontend fetches real-time price data from WebSocket and displays order book
↓
4
📝 Order Placement
User creates order (market/limit) and submits transaction to blockchain
↓
5
⚡ Order Matching
Matching engine finds counterparty and executes trade on smart contract
↓
6
✅ Settlement
Smart contract transfers tokens between parties and updates balances
↓
7
📱 UI Update
Frontend updates portfolio, transaction history, and market data

🔄 Liquidity Pool Flow

1
🌊 Add Liquidity
User deposits token pair (e.g., ETH/USDC) into liquidity pool contract
↓
2
🏷️ LP Token Minting
Contract mints LP tokens representing proportional ownership in the pool
↓
3
💱 AMM Trading
Traders swap tokens using constant product formula (x * y = k)
↓
4
💰 Fee Collection
Trading fees (0.3%) are collected and added to pool reserves
↓
5
🔄 Remove Liquidity
LP token holders can burn tokens to withdraw proportional share plus fees
